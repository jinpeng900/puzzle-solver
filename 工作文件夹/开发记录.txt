2. SOLVER ARCHITECTURE DESIGN
================================================================================
[COMPLETED] Designed solver based on allstar.cpp + algorithm document:

Core algorithm pipeline:
  1. DFS enumeration with bipartite graph pruning (upper_bound_reachable)
  2. Neighbor sorting (color/heuristic-based priority)
  3. Cache-based 2-step lookahead (now + nxt score)
  4. Sorting by (now+nxt-short_pen), keep top keep1 candidates
  5. 3-step lookahead (now + nsc + tsc + dq + bh), keep top keep2
  6. Survival check (simulate survival_steps ahead), penalty for low survival
  7. Beam Search (beam_w x beam_d depth lookahead)
  8. Return best path

Level-specific strategies (from algorithm document):
  - L1: Cascade Builder - large component first, high short_pen
  - L2: Harvest Maximizer - small component first, lower short_pen
  - L3: Bridge Connector - wildcard/bomb priority in DFS starts
  - L4: Explosive Clearer - bomb blast optimization, beam search active
  - L5: Adaptive Hybrid - 12x12 board, both wildcards+bombs

Phase detection (from algorithm document Part 3):
  - EARLY  (0-27% steps): dfs_limit * 1.25, more exploration
  - MID    (28-67% steps): balanced parameters
  - LATE   (68-89% steps): dfs_limit * 0.7, focused harvest
  - URGENT (90%+ steps):   dfs_limit * 0.5, maximize scoring
  - SURVIVAL (near-deadlock): boost surv_mul, reduce short_pen

Time management:
  - tfast: reduce params to 65% when elapsed > max_time * tfast_ratio
  - tdead: return best immediately when elapsed > max_time * tdead_ratio
